# 玩家测试 Agent 协作规范（黄金 10 分钟）

最后更新：2026-03-28
适用项目：`E:/Ai/Vibecoding_Game`

## 1. 目标与范围
- 目标：保证“前 10 分钟上手体验”稳定、清晰、可复现。
- 范围：新游戏开局到首段主线（命名 -> 领主宠 -> 教学战 -> 花冠大道 -> 家园进化实操 -> 反派战前）。

## 2. 单轮协作节奏
1. 测试 Agent 先同步测试路径（本轮覆盖哪些流程）。
2. 开发侧确认本轮关注的 P0/P1 问题与回归范围。
3. 测试 Agent 执行并产出结构化报告。
4. 开发侧修复后回传“修复点 + 自测证据 + 回归请求”。
5. 测试 Agent 只做定向回归，不重复全量扫描。

## 3. 缺陷报告模板（必须）
每条缺陷都用以下结构：
- `ID`：`BUG-xx`
- `优先级`：`P0 / P1 / P2`
- `模块`：例如 `主线`、`战斗UI`、`家园系统`
- `复现步骤`：最少 3 步，写清地图/阶段
- `期望结果`
- `实际结果`
- `证据`：截图路径、日志片段、关键变量
- `稳定复现率`：例如 `5/5`

## 4. 优先级定义
- `P0`：主线卡死、无法继续推进、存档损坏。
- `P1`：可继续玩，但体验显著受损（强误导、核心按钮不可见、战斗交互高成本）。
- `P2`：文案、轻微数值偏差、低频显示问题。

## 5. 回归门禁
- P0 必须“修复 + 回归通过”后才能进入下一轮新功能。
- 每次提交至少覆盖：
  - 新游戏主流程 1 次
  - 战斗内技能/道具/逃跑/捕捉各 1 次
  - 家园交互（泉水/仓库/吞噬或融合）各 1 次

## 6. 测试便利配置（当前已接入）
配置文件：`config/balance-config.json` -> `qa`

可用项：
- `enabled`: 是否启用 QA 便利能力
- `quickKeysEnabled`: 启用热键
- `starterKitOnNewGame`: 新开档自动发测试补给
- `battleAnimationMultiplier`: 战斗演出时长倍率（越小越快）
- `showHomeFacilitiesFromStart`: 主线前是否强制显示家园设施

默认热键（QA 开启时）：
- `F6`：发放测试补给并恢复队伍/后备/家园仓库 HP
- `I`（战斗中）：快速打开“战斗道具”选择面板

## 7. 输出约定
- 测试 Agent 输出按“发现优先、结论在后”：
  1. `P0`
  2. `P1`
  3. `P2`
  4. 通过项摘要
- 开发侧回传按“修复映射”：
  - `BUG-ID -> 修复文件 -> 验证结果`
