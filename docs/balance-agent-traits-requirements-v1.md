# 数值策划需求文档（仅特性系统）v1

## 1. 任务目标
- 仅完成“冒险特性/战斗特性”分级与数值配置，不改剧情流程。
- 建立 `B / A / S` 特性梯度，保证“稀缺但有成长感”。
- 为程序侧提供可直接接入的配置参数与验收口径。

## 2. 范围边界（必须遵守）

### IN（数值 agent 负责）
1. 特性分级标准：`B / A / S` 定义、强度区间、投放建议。
2. 特性条目设计：每条特性的效果、触发条件、边界。
3. 参数配置：维护 `config/trait-balance.json`（或同等配置文件）。
4. 平衡验证：给出“首道馆前”实测结论与调参建议。

### OUT（数值 agent 不负责）
1. 《理念之争》剧情结构、演出分镜、对白文案。
2. NPC 流程、任务触发器、地图引导实现。
3. UI 视觉与交互动效。
4. 美术资源与立绘生成流程。

说明：
- 《理念之争》可落地 v1 规格由主程侧（我这边）负责整理与接入。
- 数值 agent 只需给“特性与技能数值”输入，不改主线脚本。

## 3. 已确认硬约束
1. 仅稀有精灵/传说精灵/神兽可拥有特性。
2. 首道馆前默认不投放 S 级特性。
3. `仁慈` 仅对野生精灵生效（训练师战/道馆战/剧情 Boss 无留 1 HP 效果）。
4. 冒险特性允许“背包生效”，同类效果不叠加，默认取最高值。

## 4. 分级规则（执行版）

### B 级（基础强化）
- 强度建议：单项 `5%~12%`。
- 定位：提升手感，不改变玩法范式。
- 示例：
  - `火势增幅`：火系伤害 `+10%`
  - `奋勇`：低血增伤，上限 `+15%`

### A 级（流派核心）
- 强度建议：可明显影响战斗/探索决策，但有触发边界。
- 定位：形成流派差异，不允许无脑通吃。
- 示例：
  - `一往无前`：对低于自身等级的野生精灵，逃跑成功率 `100%`
  - `续航`：在背包/后备中按 tick 恢复 HP（建议 `1.5%~2.5%`）

### S 级（后期稀缺）
- 定位：强力上限，首道馆前仅允许“预埋不发放”。
- 示例：
  - `回赠`：野外奖励倍率 `x2`
  - `不屈`：免疫一次致命伤害并保留 `1 HP`

## 5. 交付物清单（给数值 agent）
1. `config/trait-balance.json`（完整可读，含注释说明字段含义）。
2. `docs/trait-balance-table-v1.md`（特性总表 + 边界条件 + 测试建议）。
3. 一份首道馆前实测记录（融合线/吞噬线至少各 1 局）。

## 6. 配置结构建议
```json
{
  "tierPolicy": {
    "preGymAllowSTier": false
  },
  "stackingPolicy": {
    "sameTypeStack": false,
    "priority": ["S", "A", "B"]
  },
  "traits": {},
  "mercySkill": {
    "power": 60,
    "wildOnlySpareOneHp": true
  }
}
```

## 7. 验收标准
1. B/A/S 梯度清晰，A 级可感知，S 级不过早破圈。
2. `仁慈` 在野外捕捉流程稳定生效，PVE 对战不越界。
3. 背包生效特性不出现叠加爆炸或逻辑冲突。
4. 数值改动不导致首道馆前“无脑碾压”或“必定卡关”。

## 8. 协作与提交流程
1. 数值 agent 每次只提交配置文件与数值文档，不改 `src/game/world-events.js` 等剧情文件。
2. 发现需改剧情触发条件时，仅提交“建议单”，由主程侧决定是否落地。
3. 所有重要结论同步到 `docs/context-log.md`，避免多 agent 信息断层。
