# Demo Scope

Use this file to stay inside the current contest goal.

## Timebox

- contest window: `2026-03-24` to `2026-03-29`
- active stack: Web only
- no Godot migration during this sprint

## Required Demo Loop

The demo is not done until this full path works:
- opening guide
- capture
- growth loop
- gym entry and leader battle
- victory reward
- save/load continuity

## Core Priority Rule

Prefer:
- clarity
- strategy
- progression continuity

Deprioritize:
- asset count
- map size inflation
- broad packaging that does not strengthen the loop

## What Usually Counts As In-Scope

- tutorial clarity
- route and map guidance
- battle strategy readability
- switch / bag / leveling usability
- gym gating and reward flow
- save compatibility

## What Usually Counts As Secondary

- large map expansion
- wide monster roster growth
- mass NPC portrait generation
- full content completion of side systems
- heavy presentation polish not tied to loop comprehension

## Demo Tone

The project should feel like:
- an anime-inspired monster RPG
- shaped by AI-assisted presentation
- centered on meaningful evolution choices

But the demo should prove the core game works before trying to prove the whole universe is large.
